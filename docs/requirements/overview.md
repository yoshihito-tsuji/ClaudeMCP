# Requirements Overview (2026-02-06)

## 1. 背景・目的

- GitHub公開リポジトリ（kmizu/embodied-claude）複製をベースにした別名の新規プロジェクト
- 生成AIに視覚を与えたときに起きる現象を、自室環境で学習・検証する
- 学習用途のため、配布・再現性重視ではなくローカル完結を優先する

## 2. 利用環境・前提

- 利用者: 自分のみ（自室）
- MCPクライアント: Claude Code
- 目のハードウェア: TP-Link Tapo C210
- 利用OS: macOS
- ネットワーク: ローカルLAN（固定IP推奨）
- 画像データ: 原則保存しないが、ローカル保存は許可
- 外部送信: 禁止（プライバシー保護）
- セットアップ方法: ターミナルで手動実行（VSCode上で実施）

## 3. MVPスコープ

### 機能

- wifi-cam-mcpで画像取得ができる（`see`）
- 首（パン・チルト）操作ができる（`look_left/right/up/down`）
- 最小ユースケースを確認する
  - 観察: 「今見える？」
  - 自主探索: 「見たい方向へ視線を移動」
  - 最優先: 撮影と画像の認識（Claudeによる分析）
  - 拡張: 視野方向の変更（パン・チルト）

### セットアップ

- Tapoローカルアカウントで認証
- カメラIP固定
- `.mcp.json` と `.env` で接続情報を管理

## 4. 非機能要件

- セキュリティ/プライバシー
  - 撮影データの外部送信は行わない
  - ローカル保存は許可するが、運用ルールを明確化
- 保存運用
  - 容量を圧迫する場合は削除する方針
- 可用性
  - 接続エラー時の復旧手順を明確化
- 可観測性
  - 主要操作（撮影、首操作）は最低限ログに残す

## 5. 非スコープ

- 長期記憶（memory-mcp）
- 音声認識（耳）
- 音声合成（声）
- 自律行動スクリプトの定期運用
- 再現性重視の配布・パッケージ化

## 6. 受け入れ条件

1. Claude Code から `see` が実行でき、最新の画像が返る
2. パン・チルト操作が実行でき、視点が変わる
3. 画像データが外部送信されない運用が明確である
4. 接続情報の設定手順が再現可能である
5. エラー時の最低限の復旧手順が定義されている

## 7. リスクと対策

- カメラ接続の不安定
  - 対策: IP固定、ローカルアカウント、再接続手順
- プライバシー漏えい
  - 対策: 外部送信禁止の明文化、ローカル保存方針の明確化
- 学習目的の拡散
  - 対策: MVPを「目＋首」に限定し段階的に拡張

## 8. マイルストン（目安）

1. Tapo C210到着後、初期接続と `see` 成功
2. パン・チルト操作の成功
3. 最小ユースケース（観察→回答）確認
4. 運用ルール（保存/非保存、外部送信禁止）の整理
